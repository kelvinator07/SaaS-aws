# MediNotes Pro Healthcare Consultation Assistant

AI Healthcare consultation note assistant for clinicians. Doctors paste raw visit notes and MediNotes Pro streams back a structured summary, recommended next steps, and a patient-friendly draft email — generated by GPT and rendered live as markdown.

## Features
A healthcare application that:
- Takes doctor's consultation notes as input
- Generates professional summaries for medical records
- Creates actionable next steps for the doctor
- Drafts patient-friendly email communications
- Uses structured forms with date pickers
- Streams AI-generated content in real-time


## Tech stack

**Frontend**
- Next.js 16.2.3 (pages router) + React 19.2.4
- TypeScript, Tailwind CSS v4
- `@clerk/nextjs` for auth and subscription protection
- `@microsoft/fetch-event-source` for SSE
- `react-markdown` + `remark-gfm` + `remark-breaks`
- `react-datepicker`

**Backend**
- FastAPI + Pydantic
- OpenAI Python SDK (model: `gpt-5-nano`), streaming responses
- `fastapi-clerk-auth` for JWT validation
- Deployed as a Vercel Python serverless function via the `api/` directory convention

## Prerequisites

- Node.js 18+
- Python 3.9+
- A [Clerk](https://clerk.com) account with a subscription plan keyed `premium_subscription`
- An OpenAI API key

## Environment variables

Create a `.env.local` (frontend) and/or `.env` (backend) at the project root:

```bash
# Clerk (frontend)
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...

# Clerk (backend JWT validation)
CLERK_SECRET_KEY=sk_test_...
CLERK_JWKS_URL=https://<your-clerk-frontend-api>/.well-known/jwks.json

# OpenAI
OPENAI_API_KEY=sk-...
```

## Running locally

Start the Next.js dev server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

`api/index.py` is deployed by Vercel as a Python serverless function and is automatically reachable at `/api` in production. For local FastAPI iteration you can run it directly with:

```bash
uvicorn api.index:app --reload --port 8000
```

Note that running uvicorn standalone bypasses Vercel's `/api` routing, so the frontend's `fetchEventSource('/api', ...)` call won't reach it without an additional rewrite.

## How it works

1. User signs in with Clerk on the landing page (`pages/index.tsx`).
2. On `/product`, Clerk's `<Protect>` gates the form behind the `premium_subscription` plan.
3. The form (`pages/product.tsx`) collects patient name, visit date, and notes, then calls `fetchEventSource('/api', ...)` with the user's Clerk JWT in the `Authorization` header.
4. `api/index.py` validates the JWT against Clerk's JWKS, sends the visit data to OpenAI with a system prompt that requests three markdown sections (summary, next steps, patient email), and streams chunks back as SSE.
5. The frontend buffers the stream and renders it live with `react-markdown`.

## Deployment

The repository is structured for Vercel:

- Frontend builds via `next build`.
- `api/index.py` is auto-detected as a Python serverless function.
- Set all environment variables listed above in the Vercel project dashboard.

## A note about Next.js

Per `AGENTS.md`, this project uses **Next.js 16**, which contains breaking changes versus older versions. Before editing frontend code, consult the bundled docs in `node_modules/next/dist/docs/` rather than relying on memory of older Next.js APIs.
